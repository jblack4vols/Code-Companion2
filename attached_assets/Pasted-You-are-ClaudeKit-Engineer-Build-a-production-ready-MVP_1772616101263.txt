PHI / SAFETY
- Do not store DOB.
- patient_hash = sha256(lower(name)+phone) if phone present else sha256(lower(name)+email). Do not store raw combos.
- Keep PHI out of logs.
- Emergency detection:
  If symptoms mention chest pain, shortness of breath, stroke signs, severe bleeding, suicidal ideation -> show “Call 911” banner and DO NOT schedule.

MODULE 3: UNDERPAYMENT DETECTOR (/underpayments)
- Fee schedule CRUD in /settings + CSV upload template
- Engine:
  expected_allowed = fee_schedule.allowed_expected for payer + cpt
  variance = paid - expected_allowed
  Flag if:
    - expected exists AND variance < -5
    - OR paid == 0 AND expected > 0
    - OR expected missing -> reason “missing fee schedule”
- UI:
  - Underpayments table with filters payer/location/date range/CPT/resolved
  - Totals widgets:
    - total flagged $ this month (sum of -variance where variance negative)
    - by payer
    - by CPT
  - Resolve flow: notes + mark resolved

MODULE 4: AI FRONT DESK (/frontdesk)
Build an intake workflow that feels like a real operator console:
- Left: “New Request” (form + “simulate SMS chat” input)
- Right: “Triage + Next Steps”
Steps:
1) Capture symptoms_text + location preference + contact
2) Triage (rule-based) into:
   - Red (2 hours), Orange (24 hours), Yellow (3 days), Green (7 days)
   Heuristic:
   - Red keywords: severe, unbearable, numbness, sudden weakness, fall with inability to bear weight
   - Orange: acute injury <48h, high pain, post-op concerns
   - Yellow: subacute pain limiting function
   - Green: chronic / general PT request
3) Ask for consent + insurance text
4) Offer earliest slot within acuity window for preferred location, else any location
5) Book the slot (update appointment_slots to booked, set patient_requests booked_at + appointment_time)
6) If no slots in window: add to waitlist (status=waitlist)

Scheduling for MVP:
- Seed appointment_slots for next 14 days for each location/provider
- 60-min slots on the hour from 8:00–12:00 and 2:00–5:00, lunch 1–2
- Only weekdays
